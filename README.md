# 📊 판교 vs 청라 도시공간 및 대중교통 접근성 비교분석 대시보드

이 프로젝트는 신도시 개발 모델의 대표 사례인 **판교 제1테크노밸리(삼평동)**와 **청라 국제업무지구(청라동)**를 대상으로 대중교통 접근성 및 토지이용 혼합도(LUM)를 비교 분석하는 인터랙티브 대시보드 애플리케이션입니다.

---

## 1. 프로젝트 개요
* **목적**: 두 거점 지구의 대중교통 접근성(등시간권), 배후 인구 및 종사자 구성, 토지이용 혼합도(LUM)를 정량적으로 비교하여 직주근접 및 자족도시로서의 성능을 진단합니다.
* **주요 기능**:
  * Leaflet.js 기반의 지하철 역점 30분/60분 등시간권(Isochrone) 공간 렌더링.
  * 건축물대장 및 필지 정보를 결합한 토지이용 형태 공간 매칭 및 개별 필지별 상세 용도 시각화.
  * 배후 인구/종사자수, 직주비, 토지이용 혼합도(LUM) 종합 비교 테이블 제공.
  * 주요 건축물 용도 분포 및 주요 업종 구성 Chart.js 그래프 시각화.

## 3. 분석 구역 정의 및 출처
도시 기능의 핵심 자족 용지를 명확히 비교하기 위해 다음 공간적 범위(Bounding Box)를 기준으로 분석 대상을 정의하였습니다.

1. **판교 분석 구역 (판교 제1테크노밸리 중심)**:
   * **법정동**: 경기도 성남시 분당구 삼평동 일원
   * **면적**: 약 **66만 ㎡** (Bounding Box 기준 공간 범위 필터링)
   * **공간 범위**: `[127.0980, 37.3950]` ~ `[127.1150, 37.4060]` (경도/위도)
2. **청라 분석 구역 (청라 국제업무지구 중심)**:
   * **법정동**: 인천광역시 서구 청라동 일원 (청라국제도시 개발 계획 평면 기준)
   * **면적**: 약 **177만 ㎡** (Bounding Box 및 청라 IBD 경계 다각형의 공간 조인 후 클리핑)
   * **공간 범위**: `[126.6415, 37.5325]` ~ `[126.6565, 37.5445]` (경도/위도)

## 4. 데이터 처리 과정 (`preprocess.py` 파이프라인 단계별 설명)

`preprocess.py` 스크립트는 원천 데이터(SHP, CSV)를 공간 연산하여 시각화 및 대시보드 로드용 데이터 포맷(JSON, GeoJSON)으로 가공합니다. 전체 가공 파이프라인은 다음과 같이 실행됩니다.

1. **건축물대장 PNU 매핑 및 필터링**
   * 성남시 분당구 및 인천시 서구 건축물대장 대용량 CSV 데이터를 읽어옵니다.
   * `시군구코드` + `법정동코드` + `대지구분코드` + `번` + `지` 정보를 결합하여 19자리 고유 토지코드인 **PNU**를 생성합니다.
   * 분석 구역(Bounding Box) 경계에 들어오는 유효 PNU에 속한 건축물 데이터만 필터링합니다.
2. **토지이용 필지 매칭 (Zoning Join)**
   * 연속지적도(LSMD SHP) 데이터와 VWorld 토지이용계획정보 CSV 데이터를 PNU 기준으로 매핑합니다.
   * 각 필지가 속하는 주거지역, 상업지역, 공업지역, 녹지지역 등의 도시계획 용도지역 정보를 분류합니다.
3. **SGIS 집계구 공간 조인 및 배후 통계 추출**
   * 통계청 SGIS 집계구 공간 데이터와 인구/종사자 속성 데이터를 조인합니다.
   * 분석 구역의 중심점(Buffer 중심)을 기준으로 공간 쿼리를 수행하여, 판교/청라 분석 구역 내부의 배후 인구 및 배후 종사자 수를 집계구 단위로 공간 합산합니다.
4. **등시간권 다익스트라(Dijkstra) 계산 및 버퍼 Union**
   * 수도권 지하철 네트워크(노드, 링크) 데이터를 활용해 판교역 및 청라국제도시역을 출발점으로 설정합니다.
   * **다익스트라 알고리즘**을 활용하여 대중교통 이용 시 30분 및 60분 이내에 도달 가능한 전체 지하철역 집합과 소요시간을 계산합니다.
   * 계산된 도달 가능 지하철역들을 기준으로 반경 500m 버퍼를 생성하고, 개별 버퍼들을 **공간적 합집합(Union)** 처리하여 유효 공간 영역을 추출합니다.
   * 해당 합집합 공간 영역 내부에 포함되는 SGIS 집계구 통계를 추출하여 **'등시간권 내 도달 인구 및 종사자수'**를 도출합니다. (30분/60분 통계 간 단조 증가성 검증 포함)
5. **토지이용 혼합도 (LUM) 계산**
   * 분석 구역 내 최종 필터링된 건축물대장의 총연면적을 기준으로 **엔트로피 지수(Entropy Index)** 공식을 적용해 토지이용 혼합도를 계산합니다.
   * 건축물 용도를 5대 분류(주거, 상업·업무, 공업, 공공/교육, 기타)로 분류하고 각 용도가 차지하는 연면적 비율 $p_i$를 구합니다.
   * 아래 공식에 대입해 최종 **LUM 지수**를 도출합니다:
     $$LUM = \frac{-\sum_{i=1}^{5} (p_i \ln p_i)}{\ln(5)}$$

## 5. 실행 방법

데이터 수정 또는 업데이트가 필요한 경우, 터미널 환경에서 아래 명령어를 실행하여 전처리 파이프라인을 구동할 수 있습니다.

```powershell
# 전처리 파이프라인 스크립트 실행
python preprocess.py
```

* **출력 결과물**: 스크립트 실행이 완료되면, 웹 대시보드가 참조하는 결과 JSON 및 GeoJSON 파일들이 루트 경로와 `docs/` 폴더 내에 자동으로 생성 및 갱신됩니다.
  * `docs/comparison_data.json` : 배후 인구, 종사자, 직주비, LUM, 건물 용도별 연면적 등 종합 통계
  * `docs/cumulative_accessibility.json` : 분 단위 접근 가능 누적 역 개수 데이터
  * `docs/subway_isochrone.json` : 등시간권 영역 및 도달역 정보
  * `docs/pangyo_parcels.geojson` & `docs/cheongna_parcels.geojson` : 개별 필지 경계 및 용도정보 레이어
  * `docs/pangyo_buildings.geojson` & `docs/cheongna_buildings.geojson` : 건물 외곽 정보 레이어
